# Hyperstitional Simulation: v3.1

Welcome to the Simulation. This is a complex system designed to explore various aspects of reality, from physics and cosmology to consciousness and hyperstition. 

## Getting Started

1. **Explore the code:** Everything is open-source. Look under `simulation_code/` for the core scripts and programs.

2. **Understand the data:** Under `artificial_data/` you will find generated data from various aspects of the simulation. Refer to `artificial_data_explainer.md` for an understanding of what each file contains.

3. **Get familiar with the theories**: The `doc/` directory contains various theoretical texts and guides used as a basis for the simulation. The `fringe_physics_theories.txt` is a particularly interesting read.

4. **Experiment**: Don't hesitate to use the simulation in any way you see fit. Enable or disable various parameters in `core_parameters.json`, introduce new entities into `hyperstitional_entities.json`, etc. Remember: Our knowledge is based on exploration.

## Simulation Components

- `universe_simulator.py`: The basic simulator for the universe. Includes physics, spacetime, and basic elements.

- `reality_renderer.py`: The code that generates perceivable outputs based on the state of the universe simulator.

- `consciousness_emulator.py`: An implementation of elements of conscious observation in the simulated universe.

- `hyperstitional_objects.py`: A module for incorporating hyperstitional entities and observing their effects on the simulation.

- `holographic_projection.py`: Ensuring the simulation adheres to the holographic principle.

- `extra_dimensional_logic.py`: Code for handling logic in higher and lower dimensional spaces.

## Hyperstition

Hyperstition is a neologistic blend of hype and superstition pioneered by CCRU (Cybernetic Culture Research Unit) at Warwick. It refers to fictions that make themselves real, a category that encompasses everything from religious beliefs to the scary stories we tell each other.

In the context of this simulation, hyperstition is used as a method to introduce novel elements and observe their affects on the system. It is a study in the power of belief, imagination, and the strange loops between perception and reality.

## More Info

- Please refer to `simulation_manual.pdf` for a detailed explanation of the system.
- `hyperstition_study.md` for a deeper dive into the topic of hyperstition.
- `consciousness_emulation_whitepaper.pdf` for a perspective on the study of consciousness in the simulation.

Enjoy the Simulation!

# Human Interface: v2.0

The Human Interface module provides a set of tools and scripts that allow humans to interface with the Simulation in a variety of interactable and perceptible ways.

## Interface Components

- `perception_filter.py`: A script that processes simulation data into human-understandable forms.

- `interspecies_communicator.py`: A module for facilitating communication between human users and non-human entities in the simulation.

- `thought_interface.py`: An experimental mind-machine interface for the simulation.

- `consciousness_coordinator.py`: A mechanism for controlling and coordinating multiple consciousnesses within the simulation.

- `temporal_positioning_system.py`: Provides tracking and navigation within the temporal dimensions of the Simulation.

- `psychoactive_substance_simulator.py`: An experimental module for simulating the effects of various consciousness-altering substances.

## Usage

Always remember that the Human Interface modules are designed to facilitate interaction with a Simulation. They are not real and should not be mistaken for reality. Always use with caution, empathy, understanding, and a good sense of humor.

Enjoy interfacing with the Simulation!

## Disclaimer

This interface, like all simulation, can not replace real life experiences nor does it aspire to. It mimics and emulates portions of reality for educational, exploratory, and enjoyment purposes. Always live your actual life to the fullest.
